# 状态文件模板

> 重型参考。SKILL.md 中引用的所有 `.project/` 文件的完整模板。

---

## PROJECT.md

```markdown
# [项目名称]

## 这是什么
[一段话描述项目]

## 核心价值
[必须能工作的那一件事]

## 约束
- [硬性限制 1]
- [硬性限制 2]

## 技术栈
- **语言**: [语言及版本]
- **框架**: [框架及版本]
- **数据库**: [数据库]
- **测试**: [测试框架]
- **构建**: [构建工具]

## 需求

### 已验证
- ✓ [需求] — [来源: 现有代码 | 已上线]

### 活跃
- [ ] **[分类-编号]**: [需求描述]

### 不在范围
- [排除项] — [原因]

## 关键决策

| 决策 | 理由 | 结果 |
|------|------|------|
| [选择] | [为什么] | [结果] |

---
*最后更新: [日期] [事件]*
```

## STATE.md

```markdown
# 项目状态

## 当前位置
- **模式**: [change-driven | milestone-driven | hybrid]
- **阶段**: [X / Y] — [阶段名称]（里程碑模式）
- **活跃变更**: [变更名称列表]
- **进度**: [██████░░░░] XX%

## 近期活动
| 日期 | 动作 | 详情 |
|------|------|------|
| [日期] | [动作] | [发生了什么] |

## 待决事项
- [需要决定的事] — [上下文]

## 阻塞项
- [阻塞] — [影响]

## 会话连续性
- **上次会话**: [日期]
- **停在**: [正在做什么]
- **下一步**: [建议的操作]
```

## CONFIG.md

```markdown
# 项目配置

## 工作流模式
- **模式**: interactive | yolo
- **项目模式**: change-driven | milestone-driven | hybrid
- **粒度**: coarse | standard | fine

## 质量门控
- **规划前研究**: true | false
- **计划验证**: true | false
- **执行后验证**: true | false

## 执行
- **并行化**: true | false
- **Git 追踪**: true | false
```

## ROADMAP.md

```markdown
# 路线图

## 进度

| # | 阶段 | 目标 | 状态 | 完成日期 |
|---|------|------|------|---------|
| 1 | [名称] | [目标] | ✓ 完成 | [日期] |
| 2 | [名称] | [目标] | ► 进行中 | — |
| 3 | [名称] | [目标] | ○ 待开始 | — |

## 阶段详情

### 阶段 1: [名称]
**目标:** [本阶段交付什么]
**需求:** [REQ-ID 列表]
**成功标准:**
1. [可观察的标准]
2. [可观察的标准]
```

## changes/[名称]/proposal.md

```markdown
# [变更名称]

## 做什么
[清晰描述交付内容]

## 为什么
[解决什么问题 / 增加什么价值]

## 范围
**包含:**
- [具体交付物]

**不包含:**
- [排除项] — [原因]

## 影响
- **影响文件**: [文件列表]
- **风险**: [低|中|高] — [原因]
- **依赖**: [无 | 列表]

---
*提议时间: [日期]*
```

## changes/[名称]/tasks.md

```markdown
- [ ] [任务描述]

- **总计**: N 个任务
- **已完成**: 0 个
- **剩余**: N 个

```

## debug/slug.md

```markdown
# 调试: [问题标题]

## 状态
- **阶段**: investigating | hypothesis | fixing | resolved
- **创建**: [日期]
- **更新**: [日期]

## 症状
- **预期**: [应该发生什么]
- **实际**: [实际发生什么]
- **错误**: [错误信息]
- **复现**: [复现步骤]

## 调查日志

### 假设 1: [描述]
- **支持证据**: [发现]
- **反对证据**: [发现]
- **判定**: 确认 | 排除 | 不确定

## 根因
[确认的根因，找到前为空]

## 已应用修复
[修改了什么来修复问题]
```
